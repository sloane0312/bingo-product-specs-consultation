# 知识库结构说明

本文档描述标品控标参数知识库的目录结构和查询规则。

## 知识库路径

```
/root/knowledge/
```

## 查询规则

**只读取文件名包含「控标参数」的文档**

```bash
# 查询命令
ls /root/knowledge/*控标参数*.md
```

## 当前可用的控标参数文档

| 文件名 | 产品线 | 大小 |
|-------|-------|------|
| `品高容器云平台-V1.7.0-控标参数-v20260115.md` | 品高容器云平台 | 19,514 字节 |

## 文件命名规范

控标参数文档应遵循以下命名格式：

```
{产品名称}-{版本号}-控标参数-{日期}.md
```

示例：
- `品高容器云平台-V1.7.0-控标参数-v20260115.md`
- `品高智算云平台-V1.8.0-控标参数-v20260201.md`

## 查询示例

```bash
# 读取全部控标参数内容
cat /root/knowledge/品高容器云平台-V1.7.0-控标参数-v20260115.md

# 搜索特定参数关键词
grep -n "CPU" /root/knowledge/*控标参数*.md
grep -n "Kubernetes" /root/knowledge/*控标参数*.md
grep -n "GPU" /root/knowledge/*控标参数*.md
```

## 新增控标参数文档

如需新增其他产品的控标参数文档：
1. 按命名规范创建 `.md` 文件
2. 文件名必须包含「控标参数」关键字
3. 放置于 `/root/knowledge/` 目录下
4. Agent 会自动识别并纳入查询范围
